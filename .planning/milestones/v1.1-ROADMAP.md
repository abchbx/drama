# Milestone v1.1: Routing, Memory & Integration

**Status:** ✅ SHIPPED 2026-03-20
**Phases:** 5-7
**Total Plans:** 3

## Overview

Complete the remaining Phases 5–7: Socket.IO real-time routing, memory folding engine, and end-to-end integration testing. The system now includes:
- **Socket.IO real-time messaging** with broadcast, peer-to-peer, and multicast capabilities
- **Four-layer memory management** with automated folding, token budget enforcement, and core preservation
- **Comprehensive integration testing** with 104 tests covering E2E scenarios and chaos conditions
- **LLM provider abstraction** with OpenAI and Anthropic adapters behind a unified interface
- **Formalized communication protocol** using Zod schemas, .env configuration, and structured logging

## Phases

### Phase 5: Message Routing Hub

**Goal**: Replace polling-based communication with Socket.IO-based real-time routing that prevents deadlock through heartbeat signals and timeout fallbacks.
**Depends on**: Phases 1-4
**Plans**: 1 plan

Plans:
- [x] 05-01: Socket.IO routing hub with heartbeat and timeout management

**Details:**
Socket.IO server implements broadcast, peer-to-peer, and multicast routing. Heartbeat signals confirm agent alive status periodically. Every message wait has explicit timeout with defined fallback behavior. Director has unilateral fallback when actors do not respond within timeout.

---

### Phase 6: Memory Management Engine

**Goal**: Implement automated memory folding that keeps the blackboard within token budgets while preserving climactic scenes and character voice constraints.
**Depends on**: Phases 1-5
**Plans**: 1 plan

Plans:
- [x] 06-01: Memory management engine with automated folding

**Details:**
Automated token counting on every blackboard write with 60% threshold alert. Semantic layer folding when >8K tokens, summary replaces entries. Procedural layer folding preserves character voice constraints. Core layer content is never evicted - only explicitly updated. Director can explicitly promote scenario content to core layer when plot-critical.

---

### Phase 7: Integration + Chaos Testing

**Goal**: Verify the complete system end-to-end and ensure adversarial conditions (boundary violations, silent agents, race conditions) are handled gracefully. Define the "YOLO ends here" milestone.
**Depends on**: Phases 1-6
**Plans**: 1 plan

Plans:
- [x] 07-01: Integration and chaos testing

**Details:**
End-to-end drama session test: Director + 2+ Actors complete a scene from start to finish. Chaos tests include boundary violation attempts, silent agent timeout fallback, concurrent blackboard writes, message reorder, and token budget overflow. Formalizes communication protocol with Zod schemas, .env configuration, and structured logging.

## Milestone Summary

**Key Decisions:**
- Socket.IO chosen for built-in heartbeat (prevents deadlocks)
- Semantic layer: 8K token budget threshold
- Core layer: 2K token budget, no eviction
- 60% threshold triggers token count alert (logged, not blocking)
- OpenAI and Anthropic adapters behind unified LlmProvider interface
- Zod for JSON validation, structured logger with agent attribution
