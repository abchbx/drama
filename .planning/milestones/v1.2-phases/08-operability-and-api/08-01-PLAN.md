<task type="auto" tdd="true">
  <name>Task 2: Create sessions REST routes</name>
  <files>src/routes/sessions.ts, tests/routes/sessions.test.ts</files>
  <action>
Create Express router with browser-facing session endpoints.

1. Create src/routes/sessions.ts with sessionsRouter:
   - GET /sessions - list all sessions (returns { sessions: Session[] })
   - GET /sessions/:id - get session by ID (returns Session or 404)
   - POST /sessions - create session (body: { name, sceneDurationMinutes, agentCount }, returns created Session)
   - POST /sessions/:id/scene/start - start scene (returns { status: 'running', sceneId })
   - POST /sessions/:id/scene/stop - stop scene (returns { status: 'interrupted' | 'completed' })

2. Each endpoint uses SessionRegistry for storage/retrieval
3. Error handling: 404 for not found, 409 for conflict (e.g., start on running session), 500 for server errors

Use TDD: write route tests with supertest before implementation.
