# v1.2 User Acceptance Testing (UAT) Checklist
# v1.2 用户验收测试检查清单

## Test Environment Setup / 测试环境设置

- [ ] Backend server running (`npm run dev` or `npm start`)
- [ ] Frontend server running (`cd frontend && npm run dev`)
- [ ] Mock LLM Provider configured (for testing)
- [ ] Browser opened to `http://localhost:5173` (Vite default)
- [ ] Browser console open for error monitoring

---

## 1. Complete Workflow Test / 完整工作流程测试

### 1.1 Session Creation / 创建会话

- [ ] Navigate to Sessions tab
- [ ] Click "Create Session" button
- [ ] Fill in session name (e.g., "Test Session UAT")
- [ ] Set scene duration to 5 minutes
- [ ] Set actor count to 3
- [ ] Click "Create"
- [ ] Verify session appears in sessions list
- [ ] Verify session status is "created"

### 1.2 LLM Configuration / LLM 配置

- [ ] Navigate to LLM Config tab
- [ ] Select "mock" provider (for testing)
- [ ] Set model to "gpt-4"
- [ ] Set temperature to 0.7
- [ ] Click "Save Configuration"
- [ ] Verify success toast message appears
- [ ] Verify configuration is persisted (reload page and check)

### 1.3 Session Parameters / 会话参数

- [ ] Navigate to Session Params tab
- [ ] Adjust scene duration minutes
- [ ] Adjust max tokens (e.g., 2000)
- [ ] Adjust max turns (e.g., 10)
- [ ] Click "Save Parameters"
- [ ] Verify success toast message appears

### 1.4 Start Scene / 启动场景

- [ ] Select the created session in Sessions tab
- [ ] Click "Start Scene" button
- [ ] Fill in scene details:
  - Location: "Living Room"
  - Description: "A casual conversation"
  - Tone: "Friendly"
  - Actor IDs: Select all available actors
- [ ] Click "Start Scene"
- [ ] Verify session status changes to "running"
- [ ] Verify toast message "Scene started" appears

### 1.5 Real-time Visualization / 实时可视化

- [ ] Navigate to Visualization tab
- [ ] Verify message stream is updating
- [ ] Verify communication graph is rendering nodes and edges
- [ ] Verify memory state is showing token usage
- [ ] Test pause/resume button
- [ ] Verify pausing stops message updates
- [ ] Verify resuming continues message updates

### 1.6 Dashboard Monitoring / 仪表板监控

- [ ] Navigate to Dashboard tab
- [ ] Verify session status shows "running"
- [ ] Verify agent health indicators are active
- [ ] Verify system metrics are displaying (CPU, memory, disk)
- [ ] Verify connection status shows "connected" (green)

### 1.7 Stop Scene / 停止场景

- [ ] Navigate back to Sessions tab
- [ ] Select the running session
- [ ] Click "Stop Scene" button
- [ ] Verify session status changes to "completed"
- [ ] Verify toast message "Scene stopped" appears

### 1.8 Export Script / 导出脚本

- [ ] Navigate to Export tab
- [ ] Select the completed session from dropdown
- [ ] Select "JSON" format
- [ ] Click "Export Script"
- [ ] Verify file downloads successfully
- [ ] Open downloaded JSON file and verify structure:
  - Session metadata present
  - Characters list present
  - Backbone entries present
  - Scenes array present
- [ ] Select "Markdown" format
- [ ] Click "Export Script"
- [ ] Verify file downloads successfully
- [ ] Open downloaded Markdown file and verify formatting
- [ ] Select "PDF" format
- [ ] Click "Export Script"
- [ ] Verify PDF file downloads successfully
- [ ] Open PDF and verify content is readable

---

## 2. Socket.IO Reconnection Test / Socket.IO 重连测试

### 2.1 Manual Disconnect / 手动断开

- [ ] With session running, stop backend server (`Ctrl+C`)
- [ ] Verify connection status changes to yellow ("reconnecting")
- [ ] Verify toast message "Connection lost - reconnecting..." appears
- [ ] Wait for reconnection attempts (should attempt 5 times by default)

### 2.2 Reconnection / 重新连接

- [ ] Restart backend server (`npm run dev`)
- [ ] Verify connection status changes to green ("connected")
- [ ] Verify toast message "Connection restored" appears
- [ ] Verify sessions are refetched
- [ ] Verify session state is synchronized

### 2.3 Connection Error Handling / 连接错误处理

- [ ] Try connecting to invalid backend URL
- [ ] Verify connection status shows red ("disconnected")
- [ ] Verify error message is displayed
- [ ] Try "Manual Reconnect" button (if implemented)
- [ ] Verify reconnection attempts continue

---

## 3. Error Handling Test / 错误处理测试

### 3.1 Export Error / 导出错误

- [ ] Try exporting a non-existent session ID
- [ ] Verify error message "资源不存在" appears
- [ ] Verify retry button is available
- [ ] Click retry and verify error persists
- [ ] Verify technical error details can be expanded
- [ ] Click "关闭" to dismiss error

### 3.2 Network Error / 网络错误

- [ ] Disconnect network connection
- [ ] Try creating a session
- [ ] Verify error message "网络连接失败,请检查网络设置" appears
- [ ] Reconnect network
- [ ] Retry operation and verify success

### 3.3 Form Validation / 表单验证

- [ ] Try creating session with empty name
- [ ] Verify validation error appears
- [ ] Try creating session with 0 duration
- [ ] Verify validation error appears
- [ ] Try starting scene without selecting session
- [ ] Verify error message appears
